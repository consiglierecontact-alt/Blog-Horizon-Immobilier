# Plan : Blog Horizon Immobilier — projet Astro complet

## Contexte validé
- **Brief** : PRODUCT.md rédigé (cabinet conseil investissement immo, audience investisseurs, sobre/institutionnel, tunnel SEO→lead)
- **Stack** : Astro statique + Markdown content collections, Lighthouse 100 visé
- **Palette** : Bleu profond `oklch(0.32 0.08 250)` + accent laiton `oklch(0.68 0.12 85)` sur blanc pur
- **Typo** : Fraunces (display serif) + Inter (body) + JetBrains Mono (data)
- **Scope** : Tout en une fois — design system + structure + pages + blog + 3 articles d'exemple

## Étape 1 — DESIGN.md + sidecar `.impeccable/design.json`
Rédiger le DESIGN.md complet selon le format spec (6 sections : Overview / Colors / Typography / Elevation / Components / Do's & Don'ts) + frontmatter YAML normatif + sidecar JSON avec composants (button, card, input, nav, chip, hero CTA) et tonal ramps OKLCH.

## Étape 2 — Scaffolding Astro
Créer manuellement (pas `npm create astro`, pour contrôle total) :
- `package.json` (astro 5, @astrojs/rss, @astrojs/sitemap, Fontsource ×3)
- `astro.config.mjs` (site URL, integrations rss+sitemap, markdown shiki thème sobre)
- `tsconfig.json` (strict)
- `public/favicon.svg`, `public/robots.txt`

## Étape 3 — Design tokens CSS
`src/styles/tokens.css` : variables OKLCH (palette + variations), échelle typographique clamp, spacing scale 4/8/12/16/24/32/48/64, radius 2/4/8px (très peu arrondi — institutionnel), shadows minimales (1 seule shadow subtile), easings.

## Étape 4 — Layout & SEO
- `src/layouts/Base.astro` : `<html>` `<head>` `<body>`, import tokens+base CSS
- `src/components/BaseHead.astro` : meta SEO (title, description, OG, Twitter, canonical, JSON-LD Organization), polices self-hosted Fontsource, favicon, prefers-reduced-motion

## Étape 5 — Composants partagés
- `Header.astro` : nav sticky sobre (logo texte + menu + CTA "RDV conseil"), mobile menu
- `Footer.astro` : 4 colonnes (marque, navigation, contact, légal), newsletter input
- `ArticleCard.astro` : carte article (catégorie pill, titre Fraunces, extrait, méta date/lecture)
- `CTASection.astro` : bloc conversion (RDV conseil + estimation secondaire)
- `Pill.astro` : chip catégorie
- `Figure.astro` : chiffre clé (gros chiffre Fraunces + label)

## Étape 6 — Content collections
`src/content/config.ts` : collection `posts` (title, description, pubDate, updateDate, category, readingTime, author, heroImage, tags) + collection `categories`.

## Étape 7 — Pages vitrines
- `index.astro` : hero (titre Fraunces + sous-titre + double CTA), section "Dernières analyses" (3 articles), section preuve (chiffres clés), section services résumés, CTA final
- `services.astro` : 3 piliers (stratégie patrimoniale / optimisation fiscale / sélection de biens)
- `a-propos.astro` : mission, équipe placeholders, valeurs
- `contact.astro` : formulaire de contact (nom/email/projet/message) + coordonnées
- `estimation.astro` : lead magnet (form court : situation, patrimoine, objectif)

## Étape 8 — Section blog
- `blog/index.astro` : liste paginée, filtres par catégorie
- `blog/[...slug].astro` : article complet (hero, prose typée 75ch, table des matières, auteur, CTA bas d'article, articles liés)
- `blog/categorie/[slug].astro` : articles d'une catégorie

## Étape 9 — 3 articles d'exemple (Markdown, contenu réaliste à remplacer)
1. **"Pinel 2026 : ce qui change pour l'investisseur"** (cat. Fiscalité)
2. **"Lyon, Bordeaux, Nantes : où investir en 2026 ?"** (cat. Marché)
3. **"LMNP ou SCI : quel montage pour votre premier investissement ?"** (cat. Stratégie)

## Étape 10 — Build & vérification
- `npm install`
- `npm run build` (vérifier sortie statique sans erreur)
- Lancer le dev server pour aperçu
- Indiquer commandes à l'utilisateur pour visualiser

## Livrables finaux
Projet Astro statique complet, navigable en local, prêt pour hébergement (Netlify/Vercel/Cloudflare Pages), avec :
- DESIGN.md + PRODUCT.md ancrant le craft
- ~10 pages (5 vitrines + 3 blog + home + estimation)
- 3 articles éditoriaux d'exemple
- SEO meta + sitemap + RSS
- Palette bleu/doré, Fraunces+Inter, 100% responsive, WCAG AA
